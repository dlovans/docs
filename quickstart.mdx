Get up and running with UIP's Identify API in minutes. This guide walks you through creating your first identity verification session with working code.

<Info>
**Prerequisites:** You need an API key from [uip.digital](https://uip.digital). New accounts get $20 in credits on their first business.
</Info>

## 1. Get your API key

<Steps>
  <Step title="Sign up">
    Visit [uip.digital](https://uip.digital) and create your account with UIP identity verification.
  </Step>

  <Step title="Create an API key">
    Navigate to **Settings** > **API Keys** and create a new key.
  </Step>

  <Step title="Save it securely">
    Copy the key immediately and store it in an environment variable. It is only shown once.
  </Step>
</Steps>

## 2. Create an identify session

<CodeGroup>

```bash cURL
curl -X POST https://api.uip.digital/v1/identify \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "intent": "Sign in to My App",
    "requested_data": ["first_name", "last_name", "date_of_birth"]
  }'
```

```javascript Node.js
const response = await fetch('https://api.uip.digital/v1/identify', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.UIP_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    intent: 'Sign in to My App',
    requested_data: ['first_name', 'last_name', 'date_of_birth']
  })
});

const { session_id, qr_code, expires_at } = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.uip.digital/v1/identify',
    headers={
        'Authorization': f'Bearer {API_KEY}',
        'Content-Type': 'application/json'
    },
    json={
        'intent': 'Sign in to My App',
        'requested_data': ['first_name', 'last_name', 'date_of_birth']
    }
)

data = response.json()
session_id = data['session_id']
qr_code = data['qr_code']
```

<Warning>
**`qr_code` is raw SVG markup, not a base64 PNG.** Inline it directly into your HTML — don't try to use it as an `<img src>`.
</Warning>

## 3. Display the QR code or deep link

Use device detection to show the right UX:

- **Desktop users** — inline the SVG so they can scan with the UIP mobile app.
- **Mobile users** — show an "Open UIP" button that deep-links to the UIP app via the `uip://` URL scheme.

```html
<div id="uip-desktop"></div>
<div id="uip-mobile" style="display: none;">
  <a id="uip-link" class="button">Open UIP</a>
</div>

<script>
  const isMobile = /iPhone|iPad|Android/i.test(navigator.userAgent);
  const expUnix = Math.floor(new Date(session.expires_at).getTime() / 1000);

  if (isMobile) {
    document.getElementById('uip-desktop').style.display = 'none';
    document.getElementById('uip-mobile').style.display = 'block';
    document.getElementById('uip-link').href =
      `uip://identify?sid=${session.session_id}&exp=${expUnix}`;
  } else {
    // qr_code is raw SVG markup — inline it directly.
    document.getElementById('uip-desktop').innerHTML = session.qr_code;
  }
</script>
```

Framework equivalents: `{@html session.qr_code}` (Svelte), `dangerouslySetInnerHTML={{ __html: session.qr_code }}` (React), `v-html="session.qr_code"` (Vue).

<Info>
**Session lifetime:** Sessions expire after ~5 minutes (`SESSION_EXPIRY_SECONDS` defaults to 310s). Show a timer and offer to generate a new session if it expires.
</Info>

## 4. Poll for the result

Poll `GET /v1/identify/:id` every 2–3 seconds. When the status changes to **`approved`**, the response carries the signer's identity snapshot and an `audit_id` you can use to look up the full audit record.

<CodeGroup>

```javascript Node.js
async function pollIdentify(sessionId, apiKey) {
  while (true) {
    const res = await fetch(
      `https://api.uip.digital/v1/identify/${sessionId}`,
      { headers: { 'Authorization': `Bearer ${apiKey}` } }
    );
    const data = await res.json();

    if (data.status === 'approved') {
      console.log(`Verified: ${data.signer_snapshot.target.full_name}`);
      console.log(`Audit ID: ${data.audit_id}`);
      return data;
    }
    if (data.status === 'expired') {
      throw new Error('Session expired');
    }

    await new Promise(r => setTimeout(r, 2500));
  }
}
```

```python Python
import time

def poll_identify(session_id, api_key):
    while True:
        response = requests.get(
            f'https://api.uip.digital/v1/identify/{session_id}',
            headers={'Authorization': f'Bearer {api_key}'}
        )
        data = response.json()

        if data['status'] == 'approved':
            target = data['signer_snapshot']['target']
            print(f"Verified: {target['full_name']}")
            print(f"Audit ID: {data['audit_id']}")
            return data

        if data['status'] == 'expired':
            raise Exception('Session expired')

        time.sleep(2.5)
```

That's it. No webhooks, no redirect URIs, no code exchange. The polling response is your source of truth for the signer's identity. Save the `audit_id` if you need the full audit later (AdES proof, attachments) via [`GET /v1/audit/:id`](/api-reference/endpoint/audit).

## Complete example

```javascript Node.js
const API_KEY = process.env.UIP_API_KEY;
const BASE = 'https://api.uip.digital/v1';

// Step 1: Create the session
const session = await fetch(`${BASE}/identify`, {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    intent: 'Sign in to My App',
    requested_data: ['first_name', 'last_name', 'date_of_birth']
  })
}).then(r => r.json());

console.log('Session created:', session.session_id);
// session.qr_code is raw SVG — inline into your HTML for desktop users.

// Step 2: Poll until approved or expired
let result;
while (true) {
  const poll = await fetch(`${BASE}/identify/${session.session_id}`, {
    headers: { 'Authorization': `Bearer ${API_KEY}` }
  }).then(r => r.json());

  if (poll.status === 'approved') { result = poll; break; }
  if (poll.status === 'expired') {
    throw new Error('Session expired — create a new one');
  }
  await new Promise(r => setTimeout(r, 2500));
}

// Step 3: Use the verified identity
const { full_name, date_of_birth } = result.signer_snapshot.target;
console.log(`Welcome, ${full_name}!`);
console.log(`Audit trail: ${result.audit_id}`);
```
