A **primitive** is one verification verb. Every step in a session's
[`steps[]`](/api-reference/sessions/create) names exactly one primitive plus optional
`params`. This is the per-primitive parameter reference; for the design philosophy see
[Primitives](/core-concepts/primitives).

Every wallet-backed completion verifies against the government credential in the
user's wallet. You're billed only for steps that **successfully verify**, and results arrive by
[webhook](/api-reference/webhooks) (one `step.completed` per step) — never in the create
response.

| Primitive | Verifies | Price |
| --- | --- | --- |
| [`identify`](#identify) | Government-attested identity attributes | $0.10 |
| [`age_verify`](#age-verify) | An age threshold, without revealing the birth date | $0.05 |
| [`light_sign`](#light-sign) | A signature over short inline terms | $0.15 |
| [`sign`](#sign) | A signature over one or more uploaded documents | $0.30 |

## identify

Returns selected, government-signed identity attributes. The wallet shows the user
exactly which fields are requested, and only those signed fields come back — no document
upload, no OCR.

### Params

<ParamField body="blocks" type="string[]">
  The attributes to request. **Omit** to request the default set (`first_name`,
  `last_name`, `dob`, `country`). A requested block the credential does not
  disclose is present with JSON `null` rather than failing the step.
</ParamField>

Supported blocks:

| Block | Returns |
| --- | --- |
| `first_name` | Given name |
| `last_name` | Family name |
| `dob` | Date of birth (ISO 8601) |
| `gender` | Sex/gender, when present (nullable) |
| `portrait` | Portrait image, when the wallet exposes it (nullable) |
| `document_number` | Credential / document number |
| `issue_date` | Date the credential was issued |
| `expiry_date` | Date the credential expires |
| `issuing_authority` | Authority that issued the credential |
| `country` | Issuing country selected by the verified credential route; sandbox uses fictional test data |
| `state` | Issuing jurisdiction, e.g. `US-CA` |
| `address` | Residence address (nullable) |
| `resident_state` | Residence state (nullable) |

<Note>
  `country` and `state` describe where the credential was **issued** (for example,
  `US` and `US-CA`). `address` and `resident_state` describe the holder's **residence**.
  Nullable fields (`gender`, `portrait`, `address`, `resident_state`) return `null`
  when the credential does not disclose them.
</Note>

<Note>
  ISO mDL has no single full-name element — request `first_name` + `last_name` and
  compose. `full_name` and `personal_number` are intentionally unmapped for US mDL
  (no single full-name element; no national identification number) and return
  `null`.
</Note>

### Result

`claims` — an object of the disclosed blocks, on the `step.completed`
[webhook](/api-reference/webhooks#step-completed) and via
[`GET /v1/sessions/{id}`](/api-reference/sessions/get).

```json step
{ "primitive": "identify", "params": { "blocks": ["first_name", "last_name", "country"] } }
```

## age_verify

Proves the holder meets an age threshold **without disclosing their birth date**. Alentra
reads the DOB from the verified credential, computes `age ≥ min_age`, and returns only the
boolean. The signer claims and wallet evidence are sealed in the audit (flagged redacted)
and never sent to you.

### Params

<ParamField body="min_age" type="integer" default="18">
  The age threshold to check (`age ≥ min_age`). A whole number, 1–120.
</ParamField>

### Result

`satisfied` — a boolean. No claims, ever — not even the age.

```json step
{ "primitive": "age_verify", "params": { "min_age": 21 } }
```

## sign

Produces a wallet-device signature bound to a government-attested signer identity
and one or more documents you upload. Alentra hashes each file **server-side** (you never
supply a hash — WYSIWYS), and the wallet signs the manifest. On successful
verification, the result, audit, charge, and retention promotion commit atomically.
The exact original files become immutable evidence: they are not customer-deletable
or time-purged. Only an upload that never joined a completed sign result remains
transient and cleanup-eligible.

Because it carries files, a session containing a `sign` step must be created as
**`multipart/form-data`**: a `steps` JSON part plus one file part per document. **The file
field name is the document id** the step references.

### Params

<ParamField body="document_ids" type="string[]" required>
  The ids of the documents to sign. Each must match a multipart **file field name** in
  the same request. 1–50 ids; combined upload ≤ 50&nbsp;MiB. Use opaque, non-PII
  ids because the ordered ids become part of the immutable cleartext audit proof.
</ParamField>

### Result

`document_hash` (the signed manifest digest) + `signature` + the default signer
`claims`. For a multi-file sign, the audit also keeps a per-file `{ id, hash }`
manifest, so any single file stays provable against its retained original.

<Note>
  Evidence retention is not a customer document-storage API. Keep your own originals;
  the business API returns hashes, signatures, results, and audit disclosures, not
  document-download endpoints.
</Note>

```bash multipart
curl https://api.alentra.app/v1/sessions \
  -H "Authorization: Bearer $ALENTRA_API_KEY" \
  -F 'steps=[{"primitive":"sign","params":{"document_ids":["contract"]}}]' \
  -F 'contract=@contract.pdf'
```

## light_sign

A lightweight signature over **short inline terms** — no file upload. Use it for consent,
terms acceptance, or a disclosure for which you want retained evidence. The canonical
terms text is retained in the audit, so the signed text stays self-contained.

### Params

<ParamField body="terms" type="string" required>
  The terms to sign. Up to **1000 words** (~5 minutes of reading). Alentra normalizes
  line endings, then shows and hashes that same canonical text.
</ParamField>

### Result

`document_hash` (hash of the canonicalized terms) + `signature` + the default signer
`claims`.

<Warning>
  Canonical terms are visible to the capability-URL holder and retained in cleartext
  in the immutable audit proof. Do not put secrets, payment credentials, or
  unnecessary personal data in `terms`.
</Warning>

```json step
{ "primitive": "light_sign", "params": { "terms": "I authorize a one-time charge of $49.00." } }
```

## Composition & reserved

- **Chaining:** to verify several things in one flow, list multiple steps in `steps[]` —
  the user completes them in one session and you get one webhook per step. Add a `gate` to
  a step to pause for your [continue](/api-reference/sessions/continue) /
  [stop](/api-reference/sessions/stop) decision. See
  [Create a session](/api-reference/sessions/create).
- **`envelope`** (multi-party signing — N signers across N wallets) is a **reserved**
  primitive: named in the model but **not yet available**.
