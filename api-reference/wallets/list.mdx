---
title: "List wallets"
description: "GET /v1/wallets"
---

<Note>
  `GET https://api.alentra.app/v1/wallets` · No auth required
</Note>

<Tip>
  This is a legacy provider-capability projection. Use public
  [`GET /v1/catalog`](/api-reference/catalog/get) for all new integrations; it adds
  credential profiles, country routes, rollout state, and exact runtime availability.
</Tip>

Returns the live **provider × primitive capability matrix** — every provider in this
deployment's catalog, whether its adapter is active, and which primitives an active
adapter supports. The booleans are derived from the backend registry, so the response
is the capability source of truth, but it does not prove a country/profile route is
available. Session create checks that every
primitive is available somewhere; when the hosted flow selects a wallet, Alentra checks
that adapter against the **whole chain** before issuing the first wallet request.

## Response

<ResponseField name="wallets" type="object[]">
  <Expandable title="wallet">
    <ResponseField name="id" type="string">Stable wallet id, e.g. `apple-wallet`.</ResponseField>
    <ResponseField name="label" type="string">Display name, e.g. `Apple Wallet`.</ResponseField>
    <ResponseField name="available" type="boolean">
      Whether this deployment registered an active adapter for the wallet.
    </ResponseField>
    <ResponseField name="primitives" type="object">
      Per-primitive support booleans (`identify`, `age_verify`, `sign`,
      `light_sign`).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json 200 OK
{
  "wallets": [
    {
      "id": "apple-wallet",
      "label": "Apple Wallet",
      "available": true,
      "primitives": { "identify": true, "age_verify": true, "sign": true, "light_sign": true }
    },
    {
      "id": "google-wallet",
      "label": "Google Wallet",
      "available": true,
      "primitives": { "identify": true, "age_verify": true, "sign": true, "light_sign": true }
    }
  ]
}
```
</ResponseExample>

The example shows provider adapters, not launch claims. In `SANDBOX`, the provider
filter is Google-only and `/v1/catalog` exposes Utopia as the sole available route.

See [Credential routes & coverage](/core-concepts/wallets) for context.
