---
title: "Credential route catalog"
description: "GET /v1/catalog"
---

<Note>
  `GET https://api.uip.digital/v1/catalog` · Public · No API key · No charge
</Note>

The canonical discovery ledger for jurisdiction, credential profile, presentation
provider, rollout state, and runtime availability. Use it instead of hard-coding a
country-to-wallet relationship: one country may have several routes, and one wallet
provider may present credentials from several countries.

The endpoint is public, CORS-readable, and cached for up to five minutes.

## Response

<ResponseField name="schema_version" type="string">Catalog response schema version.</ResponseField>
<ResponseField name="ledger_updated_at" type="string">Date the source-controlled coverage ledger was last verified.</ResponseField>
<ResponseField name="environment" type="string">`development`, `sandbox`, or `production`.</ResponseField>

<ResponseField name="countries" type="object[]">
  Every tracked jurisdiction. Coming-soon routes remain visible so absence and pending
  onboarding are distinguishable.
  <Expandable title="country">
    <ResponseField name="code" type="string">ISO 3166-1 alpha-2 code; `UT` is the explicit fictional Utopia sandbox jurisdiction.</ResponseField>
    <ResponseField name="name" type="string">Display name.</ResponseField>
    <ResponseField name="test" type="boolean">True only for a fictional test jurisdiction.</ResponseField>
    <ResponseField name="routes" type="object[]">
      <Expandable title="route">
        <ResponseField name="provider_id" type="string">Presentation provider, e.g. `google-wallet`.</ResponseField>
        <ResponseField name="credential_profile_id" type="string">Credential/trust profile, e.g. `utopia-id-pass`.</ResponseField>
        <ResponseField name="status" type="string">`sandbox`, `live`, or `coming_soon`.</ResponseField>
        <ResponseField name="coverage" type="string">`national`, `partial`, or `test`.</ResponseField>
        <ResponseField name="regions" type="string[]">Optional sub-jurisdictions for partial coverage, such as supported US mDL issuers.</ResponseField>
        <ResponseField name="registration_required" type="boolean">Whether provider/issuer onboarding is still required.</ResponseField>
        <ResponseField name="source_url" type="string">Official source used to verify the route.</ResponseField>
        <ResponseField name="note" type="string">A route-specific limitation, when present.</ResponseField>
        <ResponseField name="available" type="boolean">The only field that means this exact route can start in the current environment.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="providers" type="object[]">
  Provider labels plus `enabled` (allowed by this deployment) and
  `adapter_available` (transport code registered at runtime).
</ResponseField>

<ResponseField name="credential_profiles" type="object[]">
  Stable profile ids and their format, doctype, namespace, and fallback issuing
  country. Blank protocol fields mean onboarding/acceptance is incomplete and the
  profile cannot become available.
</ResponseField>

<ResponseField name="wallets" type="object[]">
  Legacy provider capability projection: provider id/label, adapter availability,
  and per-primitive booleans. Route selection must still use `countries[].routes`.
</ResponseField>

<ResponseField name="primitives" type="object[]">
  Shipped primitive ids and whether any registered provider adapter implements them.
</ResponseField>

The real response includes the full ledger, not only the Utopia excerpt above. Never
offer a route unless its own `available` value is `true`; provider adapter availability
alone is insufficient.
