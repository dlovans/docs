---
title: "Get a session"
description: "GET /v1/sessions/{id}"
---

<Note>
  `GET https://api.alentra.app/v1/sessions/{id}` · Auth: `Bearer` API key
</Note>

Returns a session's **status, progress, and the derived per-step results** already
produced — the same verdicts delivered by [webhook](/core-concepts/webhooks). The
requested claims are returned for non-anonymous primitives. `age_verify` is the one
business-redacted result: it returns only `satisfied` and `min_age`; its identity
stays sealed in the audit. Webhooks remain the push mechanism — use `GET` for
one-off reconciliation, not as a polling loop.

## Path parameters

<ParamField path="id" type="string" required>The session id.</ParamField>

## Response

<ResponseField name="id" type="string">The session id.</ResponseField>
<ResponseField name="status" type="string">
  `created` · `in_progress` · `awaiting_decision` · `completed` · `stopped` ·
  `failed` · `expired`.
</ResponseField>
<ResponseField name="current_step" type="integer">Zero-based index of the active step.</ResponseField>
<ResponseField name="step_count" type="integer">Total steps in the session.</ResponseField>
<ResponseField name="results" type="object[]">
  Results produced so far, one per completed step — the same business disclosure
  delivered by webhook. `age_verify` alone withholds its signer claims.
  <Expandable title="result">
    <ResponseField name="step" type="integer">Zero-based index of the step this verdict belongs to.</ResponseField>
    <ResponseField name="primitive" type="string">The step's primitive (`identify` · `age_verify` · `sign` · `light_sign`).</ResponseField>
    <ResponseField name="country" type="string">Non-anonymous results — route jurisdiction.</ResponseField>
    <ResponseField name="wallet" type="string">Non-anonymous results — presentation provider id.</ResponseField>
    <ResponseField name="credential_profile" type="string">Non-anonymous results — verified credential profile id.</ResponseField>
    <ResponseField name="verified_at" type="string">ISO 8601 time the verdict was produced.</ResponseField>
    <ResponseField name="audit_id" type="string">Id of the immutable audit record. Fetch the business-authorized record via `GET /v1/audits/{id}`.</ResponseField>
    <ResponseField name="satisfied" type="boolean">`age_verify` only — whether `age >= min_age`.</ResponseField>
    <ResponseField name="min_age" type="integer">`age_verify` only — the threshold checked.</ResponseField>
    <ResponseField name="claims" type="object">`identify`, `sign`, and `light_sign` — disclosed identity attributes. Signing steps request the default signer fields.</ResponseField>
    <ResponseField name="document_hash" type="string">`sign` / `light_sign` only — the hash the wallet device-signed.</ResponseField>
    <ResponseField name="signature" type="string">`sign` / `light_sign` only — the wallet signature over the hash.</ResponseField>
  </Expandable>
</ResponseField>
<ResponseField name="outcome" type="string">
  Terminal reason — `completed` · `stopped` · `failed` · `expired`. Empty while the
  session is still in progress.
</ResponseField>
<ResponseField name="client_reference_id" type="string">
  Optional — echoed back if you supplied it at create.
</ResponseField>
<ResponseField name="metadata" type="object">
  Optional — echoed back if you supplied it at create.
</ResponseField>
<ResponseField name="updated_at" type="string">ISO 8601 last update.</ResponseField>

An archived/billing-frozen business's key is rejected during authentication (`401`).
