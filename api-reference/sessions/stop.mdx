---
title: "Stop a session"
description: "POST /v1/sessions/{id}/stop"
---

<Note>
  `POST https://api.uip.digital/v1/sessions/{id}/stop` · Auth: `Bearer` API key
</Note>

Ends an active session without running its remaining steps. At a
[gate](/core-concepts/primitives#gates-pause-for-a-decision), it is the decision
counterpart to [continue](/api-reference/sessions/continue); it may also be used to
cancel a created or in-progress flow. The session moves to `stopped` and a
`session.stopped` webhook fires after persistence.

## Path parameters

<ParamField path="id" type="string" required>The session id.</ParamField>

## Behavior

- Valid for any non-terminal, non-expired session. Calling it on an existing terminal
  state is idempotent and returns that state.
- The deadline is checked first: a due session becomes `expired`, not `stopped`, and
  emits `session.expired`.
- Owner-only: the API key's business must own the session.
- Already-completed steps remain verified and billed; remaining steps are skipped.

<ResponseExample>
```json 200 OK
{
  "id": "sess_abc123",
  "status": "stopped",
  "current_step": 0,
  "step_count": 2,
  "results": [],
  "outcome": "stopped",
  "updated_at": "2026-07-13T12:02:00Z"
}
```
</ResponseExample>

`404` unknown session · `403` foreign session. See [Errors](/api-reference/errors).
