---
title: "Continue a session"
description: "POST /v1/sessions/{id}/continue"
---

<Note>
  `POST https://api.alentra.app/v1/sessions/{id}/continue` · Auth: `Bearer` API key
</Note>

Resumes a session paused at a [gate](/core-concepts/primitives#gates-pause-for-a-decision)
(`status: awaiting_decision`), advancing it to the next step. Use this when a later
step should depend on an earlier result — e.g. only proceed to `sign` once you've
checked the `identify` claims against your records.

## Path parameters

<ParamField path="id" type="string" required>The session id.</ParamField>

## Behavior

- Valid only when the session is `awaiting_decision`; otherwise returns an error.
- The deadline is checked first. A due session is persisted as `expired`, emits
  `session.expired`, and returns that terminal projection without advancing.
- A final step cannot be gated, so a successful continue always advances to a later
  step.
- Owner-only: the API key's business must own the session.

<ResponseExample>
```json 200 OK
{
  "id": "sess_abc123",
  "status": "in_progress",
  "current_step": 1,
  "step_count": 2,
  "results": [
    { "step": 0, "primitive": "identify", "audit_id": "sess_abc123_0", "claims": { "first_name": "Ada" }, "verified_at": "2026-07-13T12:01:30Z" }
  ],
  "outcome": "",
  "updated_at": "2026-07-13T12:02:00Z"
}
```
</ResponseExample>

`404` unknown session · `403` foreign session · `409` if the session isn't at a gate. See
[Errors](/api-reference/errors).
